## 1. 项目概述

### 1.1 项目背景

本项目是一个学生课程设计项目，旨在开发一款咖啡店微信小程序，实现完整的在线点单、会员管理和门店服务功能。项目采用轻量化技术栈，无需服务器部署，适合本地开发和演示。

### 1.2 项目约束

本项目作为学生项目，存在以下技术约束：

- **无服务器环境**：后端服务运行在本地Node.js环境
- **轻量级数据库**：使用SQLite文件数据库，无需安装数据库服务
- **模拟支付**：支付功能为模拟实现，不集成真实支付接口
- **本地存储**：图片等资源存储在本地文件系统

### 1.3 项目目标

- 提供流畅的在线点单体验，支持堂食/外带选择
- 建立完善的会员积分和优惠券体系
- 实现订单全流程管理和实时状态追踪
- 提供门店排队叫号服务
- 构建后台管理系统，支持商品、订单、用户管理

### 1.4 目标用户

| 用户类型 | 特征 | 需求 |
|---------|------|------|
| 年轻上班族 | 25-35岁，工作繁忙，追求效率 | 快速点单、预约取餐、会员优惠 |
| 学生群体 | 18-25岁，价格敏感，社交需求强 | 优惠活动、拼单功能、分享奖励 |
| 咖啡爱好者 | 各年龄段，注重品质和体验 | 商品详情、定制选项、积分兑换 |

## 3. 技术架构

### 3.1 整体架构

系统采用前后端分离架构，前端使用微信原生小程序开发，后端采用Node.js + Express框架，数据库使用SQLite轻量级文件数据库，图片等资源存储在本地文件系统。

### 3.2 前端架构

| 技术项 | 技术选型 |
|--------|---------|
| 开发框架 | 微信原生小程序（WXML + WXSS + JavaScript） |
| 状态管理 | MobX / 原生全局状态 |
| 网络请求 | wx.request 封装统一请求拦截器 |
| UI组件 | WeUI + Vant Weapp |
| 构建工具 | 微信开发者工具 |

### 3.3 后端架构

| 技术项 | 技术选型 |
|--------|---------|
| 运行环境 | Node.js 18.x LTS（本地运行） |
| Web框架 | Express.js |
| 数据库 | SQLite 3（文件数据库，无需服务器） |
| ORM工具 | Sequelize（支持SQLite） |
| 文件存储 | 本地文件系统（/uploads目录） |
| 支付接口 | 模拟支付（无真实支付集成） |
| 身份认证 | JWT Token |

### 3.4 SQLite数据库说明

SQLite是一种轻量级的嵌入式数据库，数据存储在单个文件中，无需安装数据库服务器，非常适合学生项目和小型应用。

**SQLite优势：**
- **零配置**：无需安装和配置数据库服务器
- **单文件存储**：整个数据库就是一个.db文件，方便备份和迁移
- **跨平台**：支持Windows、Mac、Linux
- **完整SQL支持**：支持标准SQL语法，包括事务、触发器等

### 3.5 模拟支付说明

由于本项目为学生项目，支付功能采用模拟实现，不集成真实的微信支付接口。

**模拟支付流程：**
1. 用户点击"立即支付"按钮
2. 前端显示支付中状态（模拟加载动画1-2秒）
3. 后端直接更新订单状态为"已支付"
4. 生成取餐码，返回支付成功结果
5. 前端跳转到订单详情页，显示取餐码

## 5. 接口规范

### 5.1 接口设计原则

API接口遵循RESTful设计规范，统一返回格式，接口版本控制通过URL路径实现。

#### 统一返回格式

所有接口返回统一JSON格式：
```json
{"code": 200, "message": "success", "data": {}}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 状态码，200表示成功，其他表示错误 |
| message | String | 提示信息 |
| data | Object/Array | 返回数据 |

### 5.2 用户相关接口

| 接口地址 | 方法 | 说明 |
|---------|------|------|
| /api/v1/auth/login | POST | 微信登录，传入code换取openid和token |
| /api/v1/user/profile | GET | 获取用户信息 |
| /api/v1/user/profile | PUT | 更新用户信息 |
| /api/v1/user/phone | POST | 绑定手机号 |
| /api/v1/user/points | GET | 获取积分记录 |
| /api/v1/user/coupons | GET | 获取我的优惠券 |

### 5.3 商品相关接口

## 附录

### A. 订单状态流转

```
待支付(0) → 已支付(1) → 制作中(2) → 待取餐(3) → 已完成(4)
```
或：
```
待支付(0) → 已取消(5)（超时未支付或用户取消）
```

### B. 错误码定义

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权，token无效或过期 |
| 403 | 禁止访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

- Node.js 18.x LTS
- 微信开发者工具

cd server && npm install && npm start

**小程序配置：**
在微信开发者工具中导入miniprogram目录，修改app.js中的baseUrl为本地服务地址
